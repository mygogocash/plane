# Wiki & Pages PRD — Self-Host Parity

## Executive Summary

The fork already ships the structural backbone of Plane's Wiki: a collaborative TipTap document editor, page versioning, hierarchical nesting, public/private access control, an auto-generated table of contents, and name-based global search. This PRD scopes the work to close the **PARTIAL** and **MISSING** marketed capabilities only — it does not re-build anything already PRESENT (hierarchy, versioning, access toggle, markdown/code blocks, TOC).

Per the fork strategy in `spec.md`, the plan is to (1) **unlock** CE-scaffolded collaboration (live cursors/presence) behind self-host entitlements, (2) **upgrade** thin partials (content search, bi-directional linking UI, activity feed) into first-class features, and (3) **build OPEN first-party equivalents** for the missing commercial features (page templates, inline/threaded comments, export, AI assistance, teamspaces/folders) using the existing `Page`/`PageLog`/`FileAsset` data layer, Django/DRF API conventions, and the React Router 7 + MobX web app. No proprietary Plane EE source is copied. All new server surfaces enforce workspace/project RBAC, all rich text is sanitized, and AI runs through the configured self-host provider only.

- **Page core**: `apps/api/plane/db/models/page.py` — `Page` (workspace FK, `parent` self-FK, `sort_order`, `access` 0/1, `description_json`/`_html`/`_binary`/`_stripped`, `projects` M2M via `ProjectPage`, `labels` via `PageLabel`), `PageVersion` (versioning with `owned_by`, `sub_pages_data`), `PageLog` (entity-link ledger including `back_link`/`forward_link`/`page_mention`/`user_mention`).
- **Page API**: `apps/api/plane/app/views/page/base.py` (`PageViewSet`, `PagesDescriptionViewSet`, `PageDuplicateEndpoint`), `version.py` (`PageVersionEndpoint`); routes in `apps/api/plane/app/urls/page.py` scoped `workspaces/<slug>/projects/<project_id>/pages/...`. Serializers in `apps/api/plane/app/serializers/page.py`.
- **Authorization**: `apps/api/plane/app/permissions/page.py` (`ProjectPagePermission`) — owner-only private pages, ADMIN/MEMBER create+edit, GUEST read, ADMIN delete.
- **Search**: `GlobalSearchEndpoint.filter_pages()` in `apps/api/plane/app/views/search/base.py` — `name__icontains` only; returns `name`, `id`, `project_ids`, `project_identifiers`, `workspace__slug` (no snippet, no breadcrumb path).
- **Editor**: `apps/web/core/components/pages/editor/editor-body.tsx` wires `CollaborativeDocumentEditorWithRef` from `@plane/editor`; `apps/web/ce/hooks/use-editor-flagging.ts` disables `["ai", "collaboration-cursor"]` for all three editor profiles. Live collaboration server lives at `apps/live`. Realtime mention/page events in `hooks/use-realtime-page-events.tsx`.
- **TOC**: `apps/web/core/components/pages/editor/summary/` + `navigation-pane/`.
- **Assets**: `apps/api/plane/db/models/asset.py` `FileAsset` already has a `page` FK and `PAGE_DESCRIPTION` entity type with signed-URL access — the substrate for rich media and export.
- **Entitlements**: `apps/web/ce/lib/self-host-entitlements.ts` already exposes flags `templates`, `public_views_pages`, `teamspaces`, `ai_copilot`, `audit_logs` (all `true`) via `isSelfHostedFeatureEnabled`.
- **API-key surface**: `api/v1/` → `plane.api.urls` (`apps/api/plane/api/urls/`); session surface is unversioned `/api/` → `plane.app.urls`.

## Gap to Close

- **[PARTIAL] Collaborative editor — live cursors & presence**: scaffolded in `@plane/editor` but `collaboration-cursor` is hard-disabled in `use-editor-flagging.ts`; not wired to an entitlement.
- **[PARTIAL] Global full-text search**: indexes `name` only; no content (`description_stripped`) search, no snippet preview, no breadcrumb/parent path in results.
- **[PARTIAL] Bi-directional page linking**: `PageLog` stores `back_link`/`forward_link`, but there is no backlink read API and no backlink UI panel.
- **[PARTIAL] Rich media embedding**: image + code + work-item embed exist; no Figma/Loom/generic external-URL embed extension or persistence.
- **[PARTIAL] Page activity audit trail**: `PageLog` exists but there is no consolidated activity-feed endpoint or right-sidebar "Activity" tab; comments/shares/access changes are not logged as activity.
- **[MISSING] Page templates**: no model, no API, no gallery modal.
- **[MISSING] Inline comments & discussions**: no `PageComment` model/API/UI; no text-anchored threads, resolution, or @mentions on pages.
- **[MISSING] Export & download**: no PDF/Markdown/HTML export endpoint for pages.
- **[MISSING] Space/Folder organization (Teamspaces)**: pages bind only to projects; no named space grouping with membership/visibility.
- **[MISSING] AI content assistance**: `ai` extension disabled; no summarize/generate/outline endpoints routed through the self-host provider.

## Goals / Non-Goals

**Goals**
- Ship OPEN first-party page templates, inline comments, export, and an activity feed with full RBAC and tests.
- Promote name-only search to content full-text search with snippets and breadcrumb path.
- Add a backlink read API + UI from existing `PageLog` data.
- Enable live cursors/presence behind a new `collaboration_cursor` self-host entitlement.
- Add generic external-URL (Figma/Loom) embeds with sanitization.
- Wire page AI assist (summarize/outline/generate) to the configured self-host provider only.
- Introduce a minimal Teamspace grouping for page organization that reuses existing project/workspace tenancy.

**Non-Goals**
- No proprietary Plane EE source. No premium link-graph visualization (backlink list only). No "custom branding" export theming in v1. No semantic/vector search (text full-text only). No external billing/Plane Cloud calls. No new collaboration transport — reuse `apps/live`.

## Requirements

**Functional**
- Templates: create/list/apply page templates; applying clones `description_*` + `logo_props` into a new page. Scope: workspace-global or project-scoped; visibility public/private.
- Inline comments: threaded comments anchored to a page or a text selection (anchor stored as JSON); reply, resolve/unresolve, soft-delete, @mention; reactions optional v2.
- Export: a page (optionally with descendants) to **Markdown**, **HTML**, or **PDF**; returns a signed download URL; large/recursive exports run async via the existing worker.
- Search: extend `filter_pages` to also match `description_stripped`; return a `snippet` (highlighted excerpt) and `parent_path` (breadcrumb of ancestor page names).
- Backlinks: read endpoint returning pages whose `PageLog` references the target page (`back_link`/`page_mention`).
- Activity: read endpoint returning a merged, chronological feed (page edits/version saves, comments, access changes, share/move) from `PageLog` + comment + a new lightweight `PageActivity` log.
- Live cursors/presence: when `collaboration_cursor` entitlement is on, `use-editor-flagging` removes `collaboration-cursor` from `disabled`.
- AI assist: summarize / outline / continue-writing actions; provider configured server-side; fails closed to non-AI editing when unconfigured.
- Teamspace: group pages under a named space with membership + visibility; a page may belong to a teamspace and/or projects.

**Authorization / Multi-tenant**
- Every new model carries a `workspace` FK; reads/writes filter by `workspace__slug` and (where applicable) `project_id`, mirroring `ProjectPagePermission`.
- Comments: any active project member (ADMIN/MEMBER/GUEST) on a page they can read may create a comment; only the comment author or ADMIN may edit/delete; resolve allowed for author, page owner, ADMIN, MEMBER.
- Templates/export/AI/backlinks/activity reads require the same read access as the underlying page (private pages stay owner-gated).
- Teamspace membership gates page visibility within the space; workspace ADMIN manages space settings.
- API-key (`/api/v1/`) parity is added only for read-safe surfaces (export trigger, template apply) where a contract test proves scoping; collaboration/AI stay session-only.

## Data Models

All extend `BaseModel` (UUID PK, `created_at`/`updated_at`/`created_by`/`updated_by`, soft-delete `deleted_at`), live in `apps/api/plane/db/models/`, and are registered in `models/__init__.py`.

- **`PageTemplate`** (`page_template`): `workspace` FK, `name`, `description_html`/`_json`/`_binary`/`_stripped`, `logo_props` JSON, `template_type` (`meeting_notes`/`runbook`/`charter`/`custom`), `access` (PRIVATE/PUBLIC, mirror `Page`), `owned_by` FK, optional `project` FK (null = workspace-global), `sort_order`.
- **`PageComment`** (`page_comments`): `workspace` FK, `page` FK, `actor` FK, `parent` self-FK (thread root null), `comment_html`/`_json`/`_stripped` (mirror `IssueComment` strip-on-save), `anchor` JSON (null = whole-page; else `{from,to,quoted_text}`), `is_resolved` bool, `resolved_by` FK null, `resolved_at` datetime null, `external_id`/`external_source`. Reuse the `IssueComment` sanitize-on-`save()` pattern.
- **`PageCommentReaction`** (v2, optional): mirrors `CommentReaction`.
- **`PageActivity`** (`page_activities`): `workspace` FK, `page` FK, `actor` FK, `verb` (`edited`/`commented`/`shared`/`access_changed`/`moved`/`restored`), `field` null, `old_value`/`new_value` text null, `comment` text, `epoch` float. Written by signals/service on page mutations and comment/access changes. (Reuse the existing `IssueActivity` shape conventions.)
- **`Teamspace`** (`teamspaces`): `workspace` FK, `name`, `description`, `logo_props`, `visibility` (`public`/`private`), `lead` FK. **`TeamspaceMember`** (`teamspace_members`): `teamspace` FK, `member` FK, `role`, `is_active`, unique-together `(teamspace, member, deleted_at)`. **`TeamspacePage`** (`teamspace_pages`): `teamspace` FK, `page` FK, `workspace` FK, unique constraint on `(teamspace, page)` when `deleted_at IS NULL` (mirror `ProjectPage`).
- **Extend `Page`** (no destructive change): rely on existing `description_stripped` for search; no column change required for full-text v1 (use Postgres `GIN` trigram or `to_tsvector` index added by migration, see below).

**Migration notes (forward / rollback)**
- Each model = one additive `CreateModel` migration; no column drops, no data backfill required → forward-safe, rollback = `migrate <app> <prev>` then drop tables.
- Search index: forward adds a Postgres functional index — `GIN (to_tsvector('english', coalesce(description_stripped,'') || ' ' || coalesce(name,'')))` on `pages` (or `pg_trgm` GIN on `description_stripped`). Rollback drops the index only; query falls back to `icontains`. Index creation on large tables uses `CREATE INDEX CONCURRENTLY` (migration `atomic = False`).
- `PageActivity` is purely additive logging; disabling its signal handler is the rollback for the feed without dropping the table.
- All M2M/junction tables use the `deleted_at`-aware unique constraint pattern already used by `ProjectPage`.

## API Contracts

Session routes are unversioned under `plane.app.urls` and scoped `workspaces/<slug>/projects/<project_id>/...` (matching `urls/page.py`). API-key routes mirror under `api/v1/`. All mutating routes pass `ProjectPagePermission` (or a `TeamspacePermission` for spaces).

- **Templates** (`app/urls/page.py` additions):
  - `GET/POST workspaces/<slug>/page-templates/` — list (workspace-global + project), create.
  - `GET/PATCH/DELETE workspaces/<slug>/page-templates/<template_id>/`.
  - `POST workspaces/<slug>/projects/<project_id>/pages/from-template/<template_id>/` — instantiate a page from a template (returns the new page).
- **Comments**:
  - `GET/POST workspaces/<slug>/projects/<project_id>/pages/<page_id>/comments/` — list threads, create.
  - `PATCH/DELETE .../comments/<comment_id>/`.
  - `POST .../comments/<comment_id>/resolve/` and `DELETE` to unresolve.
- **Export**:
  - `POST workspaces/<slug>/projects/<project_id>/pages/<page_id>/export/` body `{ format: "markdown"|"html"|"pdf", include_sub_pages: bool }` → `{ export_id }`; worker job produces a `FileAsset`; `GET .../export/<export_id>/` returns `{ status, url }` (signed).
- **Backlinks**: `GET workspaces/<slug>/projects/<project_id>/pages/<page_id>/backlinks/` → pages referencing this page from `PageLog`.
- **Activity**: `GET workspaces/<slug>/projects/<project_id>/pages/<page_id>/activities/` → merged feed, paginated.
- **Search**: no new route — extend `filter_pages` result objects with `snippet` and `parent_path`.
- **AI** (session-only): `POST workspaces/<slug>/projects/<project_id>/pages/<page_id>/ai/assist/` body `{ action: "summarize"|"outline"|"continue", selection?: string }` → streamed/JSON suggestion; never persists prompt; provider read from server config.
- **Teamspaces**:
  - `GET/POST workspaces/<slug>/teamspaces/`, `GET/PATCH/DELETE .../teamspaces/<teamspace_id>/`.
  - `POST/DELETE .../teamspaces/<teamspace_id>/members/<user_id>/`.
  - `POST/DELETE .../teamspaces/<teamspace_id>/pages/<page_id>/`.
- **Response envelope** follows existing DRF serializer outputs; errors are DRF `{detail}` with correct status (403 on RBAC, 400 on validation).

## Security

- **Least privilege**: reuse `ProjectPagePermission` for all page-scoped routes; private pages remain owner-only; comments author/ADMIN gated for edit/delete; teamspace membership gates space page reads; workspace ADMIN-only for space settings.
- **Signed URLs**: export artifacts are stored as `FileAsset` (private) and served via the existing signed-URL flow; never return raw object-store paths. Export links expire.
- **Sanitize rich text**: comment and template HTML pass through the same strip/sanitize path as `IssueComment`/`Page` (`strip_tags` for `*_stripped`); external embed URLs are validated against an allowlist (Figma/Loom/known hosts) and rendered in sandboxed iframes — never `dangerouslySetInnerHTML` of raw remote HTML.
- **No secret/prompt leakage**: AI routes read provider creds from server config/Secret Manager; raw prompts and API keys are never logged; AI usage logs store action + page id + token counts only, not prompt bodies.
- **Multi-tenant isolation**: search content matching is constrained to the requester's project membership (the existing `projects__project_projectmember__member=self.request.user` filter is preserved); teamspace/private pages are excluded from public/space surfaces.
- **CSRF/rate limiting**: new mutating session routes inherit existing middleware; AI and export endpoints get rate limits via the existing `rate_limit.py`.

## Edge Cases

- Template apply when source labels/states/members are missing in the target project (templates carry only document content + `logo_props`, so this is inherently safe).
- Comment anchored to text that was later deleted/edited → anchor becomes "orphaned"; show the quoted snippet and flag as "context changed" rather than mis-highlighting.
- Resolving a thread whose author was removed from the workspace.
- Export of a deeply nested page tree during worker downtime → job stays `queued`, UI polls; rollback disables the worker task by flag.
- Full-text search on huge `description_stripped` values → cap snippet length; index build uses `CONCURRENTLY` to avoid table locks.
- Live cursors when `apps/live` is down → editor must fall back to non-realtime save (fail closed).
- AI provider outage / quota exhaustion → action returns a user-facing error and the editor stays fully usable.
- Backlinks referencing a soft-deleted or moved page (`moved_to_page`) → resolve to the new page or exclude.
- Teamspace visibility change while another session has the page open.
- Mixed legacy pages created before `PageActivity` existed → feed shows version history only for those.

## Testing Strategy

TDD per `spec.md` (RED → GREEN → REFACTOR). Backend via the Docker pytest stack (`docker compose -f docker-compose-test.yml run --rm --build api-tests pytest -m unit`); frontend via Vitest; `pnpm check` for type/lint/format.

- **API contract tests** (pytest, RED first): for each new endpoint — happy path, 403 for non-member/GUEST-on-private, 400 on invalid body, workspace/project scoping (cross-workspace leakage rejected), soft-delete behavior. Comment resolve permissions; export produces a `FileAsset` and signed URL; backlinks reflect `PageLog`; template apply clones content and rejects cross-workspace template ids.
- **Search tests**: content match returns the page; snippet + `parent_path` present; non-member's pages excluded.
- **Frontend gating tests** (Vitest): `isSelfHostedFeatureEnabled` toggles each surface; `use-editor-flagging` drops `collaboration-cursor` only when the flag is on; AI action disabled when provider unconfigured; template gallery hidden when backend absent.
- **Migration checks**: forward + reverse migration runs clean on the Docker test DB; index creation is idempotent; no destructive operations.
- **AI tests**: provider mocked at the wrapper seam (mock the wrapper we own, not the SDK); fail-closed path asserted; assert no prompt body is logged.
- Coverage target ≥80% on the new modules; critical paths (RBAC, export signed URL, sanitize) require happy + failure + edge coverage.

## Milestones

Each milestone is a focused commit; backend schema milestones include reverse-migration review before deploy; worker/AI features ship behind a flag that can be flipped off.

1. **M1 — Entitlement & collaboration unlock**: add `collaboration_cursor` flag; wire `use-editor-flagging`; verify presence/cursors against `apps/live`. *Rollback: revert the flagging commit (frontend-only).*
2. **M2 — Full-text search**: extend `filter_pages` (content + snippet + parent_path) + functional index migration. *Rollback: revert query change; drop index (query falls back to `icontains`).*
3. **M3 — Backlinks read API + UI**: endpoint over `PageLog` + navigation-pane panel. *Rollback: revert; read-only, no schema risk.*
4. **M4 — Page templates**: `PageTemplate` model/API/serializer + gallery modal + store/service. *Rollback: revert UI; reverse `CreateModel` migration.*
5. **M5 — Inline comments**: `PageComment`(+`PageActivity`) models/API + comments tab. *Rollback: hide tab; reverse migrations.*
6. **M6 — Activity feed**: merge endpoint + Activity tab. *Rollback: disable signal handler + hide tab.*
7. **M7 — Export**: async worker job → `FileAsset` + ⋯-menu export. *Rollback: flag off worker task; hide menu item.*
8. **M8 — External embeds (Figma/Loom)**: allowlisted sandboxed embed extension. *Rollback: remove extension registration.*
9. **M9 — AI assist**: session endpoint through self-host provider + editor action. *Rollback: AI fails closed; revert endpoint.*
10. **M10 — Teamspaces**: `Teamspace`/`TeamspaceMember`/`TeamspacePage` + sidebar grouping. *Rollback: hide sidebar group; reverse migrations.*

## Rollback Plan

- Frontend-only milestones (M1, M3 UI, M8) roll back by reverting the milestone commit and redeploying the web image.
- Schema milestones (M2 index, M4–M7, M10) require reverse-migration review; rollback runs the reverse migration after reverting code; additive `CreateModel` tables can be dropped with no data loss to existing pages.
- Worker-backed features (M7 export, M6 activity signals) are disabled via feature flag/handler before reverting worker code.
- AI (M9) fails closed to normal non-AI editing when the provider config is removed.
- Production rollback uses a prior immutable Artifact Registry `preview-<short_sha>` tag or GKE rollout history, per `spec.md`.

## Risk Tier

**R1 (costly to reverse) for the feature set overall.** New models (`PageComment`, `PageTemplate`, `PageActivity`, `Teamspace*`) and the search index alter the schema, and export/comment routes are new public contracts — these need announce + log + reverse-migration review, but they are additive (no drops, no destructive backfill) and each ships behind a flag with a tested rollback, so they are not irreversible. Individual sub-tasks vary: M1 (entitlement flip) and M3-UI are **R2** (frontend, green-tested, just-do-it); the AI provider wiring touches secrets handling and is treated as **R1** with explicit fail-closed tests. Any route landing without RBAC + sanitize test coverage is automatically **R0** and must not merge.
