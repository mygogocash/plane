# Manut CI/CD Spec

## Executive Summary

The `preview` branch CI/CD pipeline is Manut-first. Pull requests run web and API
checks only. Pushes to `preview` run those checks, publish Manut component images
to Google Artifact Registry, run backend migrations on GKE, roll out the
`manut-app-*` workloads in `manut-ce`, and smoke `https://app.manut.xyz`.

## Current Workflow

- Workflow name: `Manut CI/CD`
- App URL: `https://app.manut.xyz`
- Artifact Registry: `asia-southeast1-docker.pkg.dev/affine-495114/affine`
- Namespace: `manut-ce`
- Helm release/service prefix: `manut-app`

Component images:

- `manut-frontend`
- `manut-backend`
- `manut-admin`
- `manut-space`
- `manut-live`
- `manut-proxy`

Image tags:

- `preview`
- `preview-<short_sha>`

## Business Goals

- Ship Manut changes to the production GCP runtime automatically.
- Remove old Plane/Railway release naming from the active fork CI path.
- Keep releases reproducible through immutable commit image tags.
- Fail clearly if GCP deploy credentials or runtime names are not configured.

## Technical Goals

- Build component images in Artifact Registry.
- Authenticate GitHub Actions to GCP with Workload Identity Federation.
- Deploy the same tag to API, worker, beat-worker, web, admin, live, and space.
- Run `./bin/docker-entrypoint-migrator.sh` before workload rollout.
- Verify the public instance endpoint after rollout.

## Requirements

- PRs targeting `preview` must run deterministic web and API checks.
- Pushes to `preview` must publish GCP Artifact Registry images after checks.
- Pushes to `preview` must deploy to GKE after image publication.
- GitHub Actions must not invoke Railway hooks or build Railway AIO images.
- Deployment must use repository variables:
  - `GCP_WORKLOAD_IDENTITY_PROVIDER`
  - `GCP_SERVICE_ACCOUNT`
- Optional repository variables may override defaults:
  - `GCP_APP_URL`
  - `GCP_ARTIFACT_REPOSITORY`
  - `GCP_PROJECT_ID`
  - `GCP_REGION`
  - `GKE_CLUSTER`
  - `GKE_NAMESPACE`

## Non-Goals

- Do not use GHCR as the production image registry.
- Do not publish to upstream DockerHub repositories from the active Manut path.
- Do not store JSON service account keys in GitHub secrets.
- Do not rename internal source package scopes or Python modules.
- Do not delete the old `plane-ce` / `plane-app` release during this pass.

## Architecture

```txt
PR to preview
  -> web checks
  -> API checks

push to preview
  -> web checks
  -> API checks
  -> build manut-* images to Artifact Registry
  -> run manut-app-api-migrate-* job
  -> kubectl set image for manut-app-* workloads
  -> wait for rollout status
  -> smoke https://app.manut.xyz/api/instances/
```

## Security

- GitHub Actions authenticates through GCP Workload Identity Federation.
- The deployer service account should have Artifact Registry writer, cluster access,
  and namespace-scoped Kubernetes rollout permissions only.
- Workflow logs must never print secret values.
- Secret values are cloned into `manut-*` Kubernetes secret names outside git.

## Edge Cases

- Missing GCP auth variables fail image/deploy jobs with an explicit error.
- Invalid Docker tag characters in branch names are normalized.
- A failed migration job prints job details and container logs before failing.
- A failed rollout stops before the smoke check.
- A newer push cancels stale runs for the same branch.

## Rollback Plan

- Keep `plane-ce` / `plane-app` running until Manut smoke passes.
- If the new CI workflow is faulty, revert the CI/CD commit.
- If Manut cutover fails, point ingress back to the old `plane-app-*` services.
- For image-level rollback on the old release, set workloads back to the previous
  `plane-*` `preview-<short_sha>` tag.
- For database issues, prefer Cloud SQL PITR clone before restoring over production.

## Testing Strategy

- `actionlint .github/workflows/*.yml`
- `git diff --check`
- `pnpm check`
- `pnpm --filter=web test`
- Workflow dispatch for `Manut CI/CD`
- Verify GAR receives `manut-*` images.
- Verify `helm status manut-app -n manut-ce`.
- Verify `kubectl get deploy,svc,ingress,certificate -n manut-ce`.
- Verify `curl https://app.manut.xyz/api/instances/`.

## Acceptance Criteria

- CI/CD no longer references Railway.
- Active CI/CD workflow name is `Manut CI/CD`.
- `preview` push builds GCP Artifact Registry `manut-*` images.
- `preview` push deploys to `manut-ce` when GCP auth variables are set.
- Deployment runs migrations before workload image updates.
- Deployment smoke proves the live Manut instance responds.
