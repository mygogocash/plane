# Plane AI (Ask / Build modes, Connectors, Semantic Actions) PRD — Self-Host Parity

## Executive Summary

The fork ships a partial Copilot: `CopilotMessagesEndpoint` (`apps/api/plane/app/views/copilot.py`, route `workspaces/<slug>/copilot/messages/` in `apps/api/plane/app/urls/external.py`) supports four modes — `answer`, `draft_subtasks`, `command`, `auto` (`COPILOT_MODES` line 37) — with keyword (`icontains`) evidence retrieval and an allowlisted action set. In-editor rephrase exists via `/api/workspaces/<slug>/rephrase-grammar/` (`apps/web/core/services/ai.service.ts` `performEditorTask`). GitHub sync models are complete; Slack has a `SlackProjectSync` model only.

This PRD closes every PARTIAL and MISSING marketed capability with open, first-party equivalents — no proprietary Plane EE source. All new surfaces route through the configured self-host LLM provider (`get_llm_config` / Vertex path already in `copilot.py`), are scoped by `workspaceSlug`/`projectId`, gated behind `ai_copilot` / `integrations` / `workflows_approvals` flags in `apps/web/ce/lib/self-host-entitlements.ts`, and fail closed to non-AI workflows when no provider is configured. We do NOT rebuild GitHub/GitLab integration (PRESENT) or the existing answer/draft_subtasks Copilot core.

The work decomposes into nine workstreams: (1) Build Mode project synthesis, (2) Ask Mode semantic upgrade, (3) Duplicate Detection, (4) One-Click Summaries, (5) AI Wiki/Brief drafts, (6) Intake auto-triage, (7) in-editor Translation, (8) Semantic Actions rule engine + audit, (9) Connectors (Slack in/out, Sentry intake) and a read/act MCP server. Agent Assignment, @mention agents, and Custom Agent ADK are scoped as a thin first-party "automation agent" layer built on the rule engine + Copilot actions — not a full proprietary ADK clone.

## Goals / Non-Goals

**Goals**
- Reach functional parity for the listed PARTIAL/MISSING AI/connector/semantic-action capabilities with open, tested, workspace-scoped backends and matching web surfaces.
- Reuse the existing Copilot LLM plumbing, action allowlist, and conversation models; extend rather than fork them.
- Every new mutating capability enforces server-side workspace/project role checks and writes to an audit trail.
- All AI features degrade gracefully (fail closed) when no LLM provider is configured.
- Feature flags default on for this self-host instance only when the capability is fully functional.

**Non-Goals**
- No port of proprietary Plane EE agent/ADK source. The "agent" is a thin rule + Copilot-action wrapper.
- No new external billing, credit metering, or Plane Cloud dependency.
- No rebuild of GitHub/GitLab sync (PRESENT) or the existing answer/draft_subtasks behavior.
- No required GPU/self-hosted embedding model — pgvector embeddings are optional with keyword fallback.
- No autonomous code execution by agents; actions remain the existing allowlist plus rule-engine action set.

## Requirements

**Functional**
1. **Build Mode** — `mode=build_project` returns `project_draft` { name, description, work_items[] (name, description, estimate, priority, labels[], assignee_suggestion), suggested_cycle }. A separate apply endpoint persists an approved/edited draft transactionally (project → issues → cycle membership). Never auto-final.
2. **Ask Mode** — when `WORKSPACE_AI_EMBEDDINGS_ENABLED` and embeddings exist, evidence retrieval ranks by cosine similarity over `IssueEmbedding`; otherwise the current `icontains` path runs unchanged.
3. **Duplicate Detection** — `POST .../issues/duplicate-check/` accepts {title, description, project_id}; returns ranked `candidates[]` { issue_id, score, matched_on[] }. Score ≥ `DUPLICATE_BLOCK_THRESHOLD` → `high_confidence=true` (client warns/blocks); below → suggestions only.
4. **Summaries** — `POST .../cycles/<id>/summarize/`, `.../projects/<id>/summarize/`, `.../initiatives/<id>/summarize/` return markdown + `rollup` { percent_complete, blockers[], at_risk[] }; optional persisted shareable summary with signed read token.
5. **AI Wiki/Brief** — `POST .../issues/<id>/generate-brief/` creates a Page (sectioned), links it to the issue, returns page_id; supports regenerate.
6. **Intake auto-triage** — on new `IntakeIssue`, an async task produces a `TriageSuggestion` { labels[], assignee, priority, project, confidence }; surfaced in review queue; applied only on human approve.
7. **Translation** — `performEditorTask` gains `task=translate` with `target_language`; routes through `rephrase-grammar` handler extended with a translate branch (no new model).
8. **Semantic Actions** — CRUD for `AutomationRule` (trigger + conditions + actions JSON); evaluated on issue lifecycle signals; executes allowlisted actions in a worker; each run logged to `AutomationRun` + audit.
9. **Agents** — `AutomationAgent` (name, scope=read_only|write, allowed_actions[]) assignable as a pseudo-assignee and `@AgentName`-mentionable in comments; mention/assignment enqueues a Copilot-action run honoring its guardrail; all actions audited.
10. **MCP server** — standalone service (`apps/mcp/` or `packages/mcp-server`) exposing tools: `create_issue`, `search_backlog`, `get_cycle_status`, `update_issue`; authenticates with a Plane personal API token; every call writes to the Plane audit trail.
11. **Slack** — settings UI to bind channels; inbound webhook → `IntakeIssue`; scheduled (daily/weekly) cycle summaries posted out; risk/overdue alerts posted to a configured channel.
12. **Sentry** — settings to register a webhook secret; inbound alert → classified issue with stack trace, severity→priority mapping, service/release link.
13. **Context-Aware Assist** — `POST .../copilot/context-assist/` accepts current `entity_type`/`entity_id`; returns blockers, at-risk items, recent changes, and suggested follow-ups with zero prior conversation.

**Authorization / Multi-tenant**
- Every endpoint resolves and validates `WorkspaceMember` (and `ProjectMember` where project-scoped) before any read or write, mirroring existing Copilot guest gating (`copilot.py` line 83).
- Write capabilities (build apply, rule mutate, agent run, summary share creation) require ≥ MEMBER; rule/agent/connector administration requires ADMIN.
- All AI generation requires `isSelfHostedFeatureEnabled('ai_copilot')`; connectors require `integrations`; rules/agents require `workflows_approvals`.
- Cross-workspace and Space/public surfaces must never expose rules, agents, connector secrets, audit logs, or AI evidence.

## API Contracts

Session-authenticated app routes under `apps/api/plane/app/urls/` (style: `workspaces/<str:slug>/…`); API-key routes under `apps/api/plane/api/` namespaced `/api/v1/`. All scope by `workspaceSlug` and, where relevant, `projectId`.

Session (`/api/`):
- `POST workspaces/<slug>/copilot/messages/` — extend `COPILOT_MODES` with `build_project`; add `context_assist` mode (or dedicated route below). Returns existing envelope + `project_draft` for build mode.
- `POST workspaces/<slug>/projects/<project_id>/build-project/apply/` — persist edited draft transactionally; ADMIN/MEMBER.
- `POST workspaces/<slug>/projects/<project_id>/issues/duplicate-check/` — {title, description} → candidates[].
- `POST workspaces/<slug>/cycles/<cycle_id>/summarize/`, `…/projects/<project_id>/summarize/`, `…/initiatives/<initiative_id>/summarize/`.
- `GET workspaces/<slug>/summaries/shared/<share_token>/` — signed-token read (no auth session), returns markdown only; never leaks IDs of private items beyond rollup.
- `POST workspaces/<slug>/projects/<project_id>/issues/<issue_id>/generate-brief/`.
- `POST workspaces/<slug>/rephrase-grammar/` — extend body with `task=translate`, `target_language`.
- `GET/POST/PATCH/DELETE workspaces/<slug>/automation/rules/` (+ `/<rule_id>/`) — ADMIN.
- `GET/POST/PATCH/DELETE workspaces/<slug>/automation/agents/` (+ `/<agent_id>/`) — ADMIN.
- `GET workspaces/<slug>/intake/<intake_id>/triage-suggestions/`, `POST …/<suggestion_id>/apply/`.
- `POST workspaces/<slug>/copilot/context-assist/` — {entity_type, entity_id}.
- Settings: `…/integrations/slack/channels/`, `…/integrations/sentry/`.

Webhooks (unauthenticated, signature-verified, rate-limited):
- `POST workspaces/<slug>/integrations/slack/events/` — Slack signature verified; inbound → IntakeIssue.
- `POST workspaces/<slug>/integrations/sentry/webhook/` — HMAC against `SentryProjectSync.webhook_secret`.

API-key (`/api/v1/`, for MCP server):
- `POST /api/v1/workspaces/<slug>/projects/<project_id>/issues/` (reuse existing), `GET …/issues/?search=`, `GET …/cycles/<id>/status/`, `PATCH …/issues/<id>/` — all token-scoped, audited.

Response envelope follows existing Plane DRF conventions; AI endpoints return `{ "error": "LLM provider … required" }` with 400 when `is_llm_configured` is false (mirrors `copilot.py` line 92).

## Security

- **Least privilege** — reads require workspace/project membership; AI writes ≥ MEMBER; rule/agent/connector admin = ADMIN. Guests blocked from all write modes (extend existing `WRITE_MODES` check). Agent `read_only` scope physically cannot invoke write actions (validated server-side, not UI-only).
- **Signed URLs / tokens** — shared summary links use a random, expiring `share_token`; the shared read endpoint returns rollup markdown only, never private item IDs or unsanitized content.
- **Sanitize rich text** — AI-generated brief/wiki content and any imported connector content (Slack messages, Sentry payloads) are sanitized before persistence and render; no raw HTML injection into pages/comments.
- **No secret/prompt leakage** — never log LLM API keys, Slack/Sentry secrets, session tokens, or full prompts containing workspace data; webhook secrets stored via Secret Manager/K8s secrets per spec.md, never committed.
- **Webhook hardening** — Slack signature verification and Sentry HMAC mandatory; reject unsigned/replayed requests; rate-limit inbound webhook and duplicate-check endpoints.
- **MCP** — API-key scoped to a single user's workspace permissions; tool calls cannot exceed the key holder's role; every call audited.
- **Fail closed** — when no LLM provider configured, all AI endpoints 400 cleanly and UI shows a disabled state; non-AI flows (manual create, manual triage) remain fully functional.

## Edge Cases

- Build draft references labels/states/members that don't exist in the target project → create-or-skip with a per-item warning; never fail the whole apply.
- Duplicate-check on a brand-new project with empty backlog → returns empty candidates, never blocks.
- Embeddings absent or stale (content_hash mismatch) → silent fallback to keyword retrieval.
- Summary requested on an empty cycle/project → returns "no activity" markdown, zeroed rollup.
- Translation of an empty selection or unsupported language → clear validation error, no destructive replace.
- Rule loops (rule A triggers an event that re-triggers rule A) → per-event execution depth cap + idempotency guard in `AutomationRun`.
- Agent removed/deactivated while assigned or mid-run → run aborts, audit notes "agent unavailable," assignment cleared gracefully.
- Slack/Sentry webhook arrives for a channel/project no longer bound → ignored, logged at info, not 500.
- LLM/Vertex quota exhaustion mid-generation → 503 with retry hint; partial drafts never persisted.
- Concurrent build-apply for the same draft → transactional; second apply is a no-op idempotent on draft token.
- Self-host instance with `ai_copilot=true` but provider unset → flag on, endpoints fail closed (mirrors spec.md AI requirement).

## Rollback Plan

- Every milestone is a single focused commit; frontend-only regressions roll back by reverting the commit and redeploying web.
- Schema milestones require reverse-migration review before deploy; reverse migrations only drop the new table(s) and never alter `IntakeIssue`, `SlackProjectSync`, Page, or Copilot tables.
- Background-worker features (rule engine, triage classifier, scheduled Slack summaries) are disabled by feature flag / `is_active` before reverting worker code.
- All AI features fail closed to existing non-AI workflows when the provider is unavailable, so disabling `ai_copilot` is itself a safe partial rollback.
- The MCP server is a separate deployable artifact; it can be removed without touching the Plane API.
- Production code rollback uses a prior immutable Artifact Registry `preview-<short_sha>` tag or GKE rollout history, per spec.md.

## Risk Tier

**R1 — costly to reverse.** Justification: the work adds new API contracts, new schema (including the pgvector extension), connector webhooks, and an automation engine that performs writes against live workspace data — these alter behavior teams will build on and are not trivially reversible like a copy edit. It is not R0 because there is no money movement, no production-data destruction, and no public-API contract break on existing endpoints (all changes are additive/backward-compatible, gated, and fail-closed). Per the workrules, untested critical paths (build-apply transaction, agent write guardrails, rule-loop safety, webhook signature verification) are treated as R0 individually and MUST land with full test coverage before deploy; absent that coverage, those specific paths block. Announce + log + document the rollback path for each schema/connector milestone before rollout.
